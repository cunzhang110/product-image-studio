# 实拍质感优化设计

## 目标

通过“生成前真实摄影约束 + 生成后浏览器本地轻量处理”提升图片的自然实拍观感。该功能不用于规避平台检测，不承诺改变平台对 AI 内容的判断。

## 生成前约束

在提示词生成请求中加入固定摄影真实性规则，并要求规则进入每条最终生图提示词：

- 使用符合现实世界的单一主光源和合理补光。
- 产品与桌面、手部或道具之间必须有方向一致的接触阴影。
- 反射、透明度、材质高光、景深和透视符合真实镜头规律。
- 避免过度 HDR、过度锐化、塑料质感、无瑕表面、悬浮和不合理光晕。
- 允许轻微曝光偏差、自然景深误差和手机摄影质感，但不得损害产品、标签、Logo 和文字。
- 产品参考图仍是产品外观的唯一权威来源。

该约束同时适用于多场景和同场景提示词方案。

## 生成后本地处理

新增独立的本地图像处理模块，使用 Canvas 完成可重复、可测试的轻量调整：

- `off`：不处理。
- `subtle`：轻微降低数字锐利感，加入极弱自然颗粒，并做保守的色调调整。
- `natural`：在 `subtle` 基础上增加轻微暖色白平衡、柔和高光和适度局部对比度。

处理规则：

- 保持原像素尺寸和画幅，不裁剪、不缩放。
- 不做几何变形，不修改产品结构、标签、Logo 或文字。
- 不调用任何 AI 或供应商 API，不增加生成费用。
- 处理失败时返回原图并显示错误提示。
- 不复用旧 `processAntiAI()` 的裁边、高强度噪点和“防检测”命名。

## 数据模型

批次新增 `photoFinishLevel: "off" | "subtle" | "natural"`，新批次默认 `subtle`。旧批次归一化为 `subtle`。

图片任务同时保存：

- `resultUrl`：供应商原始结果，始终保留。
- `finishedResultUrl`：本地优化结果，可选。

优化结果不覆盖原图。失败重试或重新生成图片时清空旧的 `finishedResultUrl`。

## UI 与流程

- 执行设置增加“实拍质感”三档选择：关闭、轻微、自然。
- 图片生成成功后按批次设置自动执行本地优化。
- 结果卡允许切换“原图 / 实拍优化”。
- 下载单图和 ZIP 默认使用当前可用的优化图；没有优化图时使用原图。
- 用户可重新选择档位并对已完成结果执行“重新优化”，不重新调用生图服务。

## 测试

- 提示词请求包含真实性规则，覆盖多场景与同场景。
- `off` 原样返回；其他档位保持尺寸且输出新数据 URL。
- 处理错误回退原图。
- 任务原图不被覆盖，重试时清除旧优化图。
- 下载优先选择优化结果。
- IndexedDB 旧批次归一化兼容。
- 桌面和 390px 移动端无横向溢出，控制台无错误。

## 非目标

- 不保证或测试绕过任何平台的 AI 检测。
- 不移除来源标识，不伪造相机元数据。
- 不做 AI 二次修图。
- 不自动修复手指、文字、瓶型或复杂光影错误。
