# 设置模式与提示词模板偏好设计

## 目标

优化产品设置页的信息层级，并减少每次新建产品批次时重复填写模板的操作。

## 操作模式布局

- 将“手动 / 自动”从当前双列设置网格中移出。
- 在产品名称和其他生成设置之前，使用一整行独立的操作模式分段选择区。
- “多场景创意 / 同场景多机位”继续作为单独的生成方式设置，位于操作模式之后。
- 桌面端保持横向清晰分组，移动端改为纵向排列，按钮文字不得截断或重叠。
- 主场景生成完成后，现有的模式锁定规则保持不变。

## 产品图名称联动

- 上传、拖入或更换产品参考图时，从文件名移除最后一个扩展名，并用结果更新产品批次名称。
- 只有 `nameSource` 为 `automatic` 的批次允许自动更新名称。
- 用户手动编辑批次名称后，`nameSource` 变为 `manual`；之后更换产品图不得覆盖手动名称。
- 空文件名或只有扩展名时不修改当前批次名称。

## 默认提示词模板

首次使用且没有已保存模板偏好时，新批次使用以下默认模板：

> 保留酒瓶产品，并确保酒瓶完整清晰地出现在画面中。酒瓶不能缺失、细节清楚。场景替换为【XXX】。使用 iPhone 后置镜头拍摄，符合现实世界逻辑，呈现自然、透亮、生活化的日常快照质感，风格简约、松弛、真实。

- 新批次的创作引导默认为空字符串。
- 已存在的批次不因本功能上线而被覆盖或迁移模板内容。

## 模板偏好记忆

- 用户修改当前批次的提示词模板时，同时将完整文本保存为全局模板偏好。
- 该偏好只影响修改之后新建的批次。
- 其他已有批次保持各自模板内容，不同步更新。
- 用户将模板清空也视为有效偏好；之后新建批次保持空模板，直到用户再次修改。
- 偏好使用现有 IndexedDB `settings` 存储，并与批次数据分开。
- 页面初始化时先读取模板偏好；读取失败时使用内置默认模板，不阻断应用启动。

## 数据流

1. 应用启动时加载批次数据和全局模板偏好。
2. 没有已保存偏好时，以内置默认模板作为当前新批次模板。
3. 新建或重置唯一批次时，将当前全局模板偏好传入批次创建函数。
4. 用户编辑模板时，先更新当前批次，再异步保存全局偏好。
5. 产品图处理成功后，沿用现有文件名联动函数更新自动名称。

## 验证

- 单元测试覆盖内置默认模板、显式模板覆盖、产品文件名联动和手动名称保护。
- 存储测试覆盖模板偏好的保存、读取和空字符串偏好。
- 浏览器验收桌面与移动端模式布局、模板默认值、模板修改后新建批次继承、旧批次不变，以及产品图文件名联动。
- 完整运行测试、TypeScript 类型检查和生产构建。
