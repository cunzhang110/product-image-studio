# Muzhi 多产品批次调度设计

## 目标

让多个使用 Muzhi 的产品批次同时生图，同时保证每个批次内部严格按图片编号顺序执行，并把全部批次的实际生图并发控制在安全范围内。

## 核心规则

- Muzhi 全局生成并发默认值为 `7`，允许设置 `1` 至 `10`。
- 并发值是所有 Muzhi 产品批次共享的总上限，不是每个批次各自的上限。
- 单个产品批次任何时刻最多只有一张图片处于实际生成请求中。
- 同一批次必须等待前一张结束后，才允许派发下一张，因此单批次严格保持 `1 → 2 → 3` 顺序。
- 多批次使用轮询公平调度。例如 A、B、C 同时运行时，先派发 A1、B1、C1；任一名额释放后，再按轮询顺序派发该批次的下一张。
- 如果只有 3 个活动批次，即使全局并发设为 7，实际并发也只有 3；不得为了占满 7 个名额而并发执行同一批次的多张图片。
- 新启动的批次加入队尾，不得终止或重置正在运行的其他批次。
- 停止操作只影响当前产品批次；其他批次继续运行。
- 已完成图片始终保留；恢复批次时只重新排入未完成任务。

## 范围

- 本次多批次并发只对 Muzhi 生图任务生效。
- 云雾和 APIMart 保持现有串行与限流行为，不因本次修改提高并发。
- 提示词生成不进入 Muzhi 生图调度器；OpenRouter 调用保持现有逻辑。
- 主场景生成、主场景确认、同场景分支和多场景创意最终都通过同一 Muzhi 调度器派发生图任务。

## 调度器

新增独立的 Muzhi 批次调度模块，职责包括：

- 接收以 `batchId` 分组的图片任务。
- 为每个批次维护 FIFO 队列和最多一个运行中任务。
- 在全局空闲名额内按批次轮询派发。
- 返回每个批次自己的完成 Promise 和进度更新。
- 支持按 `batchId` 取消排队任务，并向当前运行请求发送该批次的 `AbortSignal`。
- 批次取消、失败或完成后释放名额并继续调度其他批次。
- 同一批次重复加入时，不得重复派发相同任务 ID。

Muzhi 原有的单请求全局锁需要移除或绕过，否则全局并发设置无法生效。HTTP `429` 重试、`Retry-After` 和错误转换逻辑继续保留；并发上限由新调度器统一控制。

## 运行控制

- 用按 `batchId` 保存的运行控制器替换当前唯一的全局 `AbortController`。
- 运行状态必须按批次派生，不能再用一个全局 `imageRunning` 阻止其他批次启动。
- 切换产品批次不会改变后台运行状态。
- 当前批次的停止按钮只能获取并终止当前批次对应的控制器。
- 页面卸载时终止所有仍在运行的批次。

## 界面

执行设置中，选择 Muzhi 时显示：

- `Muzhi 全局并发`
- 当前值和上限，例如 `7 / 10`
- 当前实际生成数
- 当前排队任务数
- 当前运行批次数

产品批次列表继续使用批次状态标记，并明确区分：

- `排队中`
- `生图中 2/5`
- `已完成`
- `部分完成`
- `已停止`

切换到一个正在运行的批次时显示该批次的停止按钮；切换到其他未运行批次时仍可启动，不显示对其他批次的停止操作。

## 并发偏好

- Muzhi 全局并发值存入现有 IndexedDB `settings`。
- 首次使用默认 `7`。
- 旧数据没有该字段时迁移为 `7`。
- 用户调整后即时影响后续派发，不强行中断已经在运行的任务。
- 如果从较高数值调低，当前运行任务自然结束；新任务等待，直到实际运行数低于新上限。

## 失败与恢复

- 单张图片失败只标记该图片失败，不阻塞其他批次。
- Muzhi 返回 `429` 时沿用现有重试与冷却规则；冷却不应把任务错误标为停止。
- 用户主动停止时，当前批次的排队和运行任务标为 `stopped`，其他批次不受影响。
- 恢复后按当前队列公平顺序重新加入。
- 调度器异常必须释放占用名额，避免队列永久卡死。

## 验证

自动化测试必须覆盖：

- 10 个以上批次排队时，实际并发从不超过 `7`。
- 每个批次同一时刻最多一个任务运行，并保持编号派发顺序。
- 多批次按轮询顺序获得名额，不发生单批次长期独占。
- 启动新批次不会中止旧批次。
- 停止 A 不影响 B、C，A 已完成结果保留。
- 全局并发从 7 调低后不终止运行任务，只暂停新派发。
- Muzhi 绕过旧单请求锁后可以观察到多个并行生图调用。
- 云雾和 APIMart 仍保持旧行为。
- 切换批次时，各批次按钮和状态互不串扰。

浏览器验收桌面与移动端的并发设置、批次状态和停止按钮。真实 Muzhi API 只做受控小规模冒烟测试；并发上限 `7` 主要通过可重复的模拟延迟请求验证，避免无必要地产生 7 张付费图片。
